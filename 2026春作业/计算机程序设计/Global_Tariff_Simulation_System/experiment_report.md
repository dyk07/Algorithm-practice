# 全球关税模拟系统

## 功能的代码位置及实现思路
- **系统初始化与数据持久化**：`TariffSystem` 类的 `__init__`，` save_data`，` load_data` 方法 37-89 行
  初始化国家列表、商品列表、关税信息等数据结构，并设置默认值。将数据从tariff_data.json 文件加载到系统中，或者在文件不存在时创建一个新的数据文件。后续数据将始终保存在 tariff_data.json 文件中，确保数据的持久化。
- **国家管理**：`TariffSystem` 类中的 `add_country`, `delete_country`, `update_country` 等方法 91-135 行
- **商品管理**：`TariffSystem` 类中的 `add_commodity`, `delete_commodity`, `set_production_price`, `cancel_production` 等方法 137-192 行
- **关税管理与矩阵打印**：`TariffSystem` 类中的 `set_tariff`, `get_tariff` 等方法 194-248 行
- **指定路径价格计算**：`TariffSystem` 类中的 `calculate_path_price` 方法 250-275 行
- **最优采购路径查询**：`TariffSystem` 类中的 `find_optimal_purchase_path` 方法 277-342 行
  采用了经典的 Dijkstra 算法来计算最短路径，考虑了生产价格、关税费用和过境费。
- **制造业回流分析**：`TariffSystem` 类中的 `analyze_manufacturing_retrogression` 方法 344-395 行
  算法为：{
  1. 入队：将所有生产国的 (出厂价, 0跳, 路径列表) 放入堆。
  2. 循环：
   取出当前最便宜的路径。
   终点检查：如果当前国家是目的国，直接返回结果（此时必为最优）。
   剪枝：判断当前路径是否比历史记录更差，是则跳过。
   扩展：遍历邻居国家，根据公式 $P_{next} = P_{curr} \times (1 + \text{关税}) + \text{过境费}$ 计算新价格，并增加跳数重新入队。
  3. 结束：若队列为空仍未达终点，则无解。
  }
  该算法的时间复杂度为 $O(E \log N)$，其中 N 是国家数量，E 是边的数量。
- **撤销功能**：`TariffSystem` 类中的 `undo_tariff_change` 等方法 397-444 行
  通过维护一个历史记录栈来实现撤销功能，每次进行关税调整时将当前状态保存到栈中，用户可以通过调用 `undo` 方法来回滚到上一个状态。需要注意的是，历史记录仅保存在运行时内存中，程序重启后将无法数据回滚。
- **数据导入导出功能**：`TariffSystem` 类中的 `export_data` 和 `import_data` 方法 446-484 行
  允许用户将当前系统数据导出为 JSON 文件，或者从 JSON 文件导入数据（与 tariff_data.json 文件格式一致）。这使得用户可以轻松备份和恢复数据，或者在不同环境之间迁移数据。
- **图形用户界面**：`TariffSystemGUI` 类 487-1110 行
  使用 Tkinter 库构建图形用户界面，提供国家管理、商品管理、关税设置、路径查询和制造业回流分析等功能的交互界面。界面设计简洁，用户友好，支持数据的可视化展示和交互操作。
- **命令行界面**：`TariffSystemCLI` 类 1113-1384 行
  提供一个基于命令行的交互界面，允许用户通过文本输入来管理国家、商品、关税以及进行路径查询和制造业回流分析。适合喜欢使用命令行工具的用户。

注：各数据级联已在代码中正确实现（CRUD），其正确性已通过测试验证。具体过于复杂，不再赘述。

## 测试样例和测试过程
- 创建了 generate_test_data.py 脚本来生成测试数据，包括国家列表、商品列表、关税信息等。通过运行该脚本，可生成一个包含 50 个国家和 300 个商品的测试数据集，并随机分配关税和生产价格，确保测试数据的多样性和真实性。

具体过程如下：
1. 运行 generate_test_data.py 脚本生成测试数据。
2. 运行 tariff_system.py 脚本启动图形用户界面。
3. 使用生成的测试数据进行功能测试。
   - 在国家管理界面，添加、删除和更新国家信息，验证数据的正确性。
   - 在商品管理界面，添加、删除商品，设置生产价格和取消生产，验证数据的正确性。
   - 在关税管理界面，设置和查询关税信息，验证数据的正确性。
   - 在路径价格计算界面，输入不同的路径和商品组合，验证路径价格计算的正确性。
   - 在最优采购路径查询界面，输入不同的国家和商品组合，验证最优采购路径的计算结果。在当前规模下，计算结果可在 0.5 秒以内返回，证明了算法的高效性。
   - 在制造业回流分析界面，输入不同的本国和商品组合，验证分析结果的合理性。在当前规模下，计算结果可在 0.5 秒以内返回，证明了算法的高效性。
   - 注：
   (1) 图形界面的显示均正常，各表格可以正确大量数据，并且界面元素的布局合理，用户操作流畅。
   (2) 以上测试过程也在命令行界面中进行了验证，确保了两种界面的一致性和功能的正确性。
4. GUI 右上角关闭键与 CLI 的退出命令均可正常关闭程序。

## 作业要求的测试样例
- 已在 test_data.json 文件中提供了一个测试样例
- chip 在 chn > vnm > sgp > jpn > usa 的路径价格为104.00，而在 chn > usa 的路径价格为 201.00
- 通过对usa芯片制造业回流分析，得到关税调整建议（仅提高直接进口关税）:

## 创新功能实现
- 图形用户界面：使用tkinter等标准库为程序设计图形界面。
- 任意跳数支持：用户可以自行选择跳数（默认为4）；如果将跳数设为总国家数即相当于考虑所有路径，通过Dijkstra算法可以高效找到最优路径（见上文）。
- 数据导入导出功能：在TariffSystem类中新增了`export_data`和`import_data`方法，允许用户将当前系统数据导出为JSON文件，或者从JSON文件导入数据（与tariff_data.json文件格式一致）。这使得用户可以轻松备份和恢复数据，或者在不同环境之间迁移数据。并同时在图形界面（关税管理界面底部）和命令行界面（7.系统功能）中添加了相应的按钮和命令来调用这些功能，增强了系统的实用性和用户体验。

其他创新功能：
- 制造业回流分析结束后可一键调整关税，并可以一键撤销关税调整，极大提升了用户的操作便利性（在GUI和CLI中均支持）。当然，当数据量极大时，由于需要保存历史记录并支持回滚，可能会耗时几秒。
- 在底栏增加状态显示，实时反馈用户操作结果和系统状态，提升用户体验。
